# Trove Extension — Safari Setup (macOS + iOS)

Safari requires web extensions to be wrapped in a native app container built with Xcode. Apple's `safari-web-extension-converter` tool generates the Xcode project from the shared extension source, and you then build and run it like any native app.

## Prerequisites

- **Xcode** installed with command-line tools (`xcode-select --install`)
- **Apple Developer account** — a free account works for personal use (sideloading to your own devices)

## Generate the Xcode Project

Run the following command from the repository root:

```bash
xcrun safari-web-extension-converter extension/shared \
  --project-location extension/safari \
  --app-name "Save to Trove" \
  --bundle-identifier com.trove.save-extension \
  --swift
```

This creates an Xcode project at `extension/safari/Save to Trove/Save to Trove.xcodeproj`.

> **Note:** The generated Xcode project is gitignored. You will need to re-run this command after cloning the repository or if the extension source in `extension/shared/` changes.

## macOS

1. Open `extension/safari/Save to Trove/Save to Trove.xcodeproj` in Xcode.
2. Select the **macOS** target in the scheme selector (top-left of the Xcode toolbar).
3. Configure signing: go to **Signing & Capabilities** and select your development team.
4. Click **Product → Run** (or press `Cmd+R`). This builds the container app and launches it.
5. Open **Safari → Settings → Extensions** and enable **Save to Trove**.
6. Click the extension icon in Safari's toolbar to open the popup, then configure your server URL and API token in Settings.

### Allow Unsigned Extensions (Development)

If you are using a free Apple Developer account, you may need to enable unsigned extensions:

1. In Safari, go to **Settings → Advanced** and tick **Show features for web developers**.
2. A new **Developer** menu appears in the menu bar. Select **Developer → Allow Unsigned Extensions**.
3. This setting resets each time Safari is relaunched, so you will need to re-enable it after each restart.

## iOS

1. In the same Xcode project, select the **iOS** target in the scheme selector.
2. Configure signing for the iOS target: go to **Signing & Capabilities** and select your development team.
3. Connect your iPhone via USB, or select a simulator (e.g. iPhone 17).
4. Click **Product → Run** (or press `Cmd+R`). Xcode builds and installs the container app on your device.
5. On your iPhone, go to **Settings → Apps → Safari → Extensions** and enable **Save to Trove**.
6. In Safari, tap the puzzle piece icon in the address bar to access the extension popup, then configure your server URL and API token.

## Configuration

The configuration process is the same as Chrome once the extension is running:

1. Open the extension's **Settings** page (accessible from the popup).
2. Enter your **Trove server URL** (e.g. `https://trove.your-tailscale-domain`). Do not include a trailing slash.
3. Enter your **API token**.
4. Tap **Test Connection** to verify, then **Save**.

## Usage

- **Toolbar popup** — click (macOS) or tap (iOS) the extension icon to save the current page with a title, collection, and tags.
- **Context menu** (macOS only) — right-click on any page or link and select **Save to Trove**.
- **Keyboard shortcut** (macOS only) — press `Cmd+Shift+L` to open the popup.

> **iOS note:** Context menus and keyboard shortcuts are not available in Safari on iOS. Use the toolbar popup to save links.

## Notes

- The **container app** generated by the converter is minimal — it displays instructions for enabling the extension in Safari. The actual extension functionality lives entirely in the Safari Web Extension target.
- **Rebuilding:** If you update the source files in `extension/shared/`, you can either regenerate the Xcode project with the converter command above, or manually replace the extension resources inside the existing project under the `Resources/` folder.
- **Distribution:** For sharing the extension beyond your own devices, submit the container app to the App Store or distribute via TestFlight. Ad-hoc and enterprise distribution also work for internal use.

## Troubleshooting

| Problem | Solution |
| ------- | -------- |
| Extension does not appear in Safari settings | Ensure you ran the container app at least once. Safari only registers extensions from apps that have been launched. |
| "Allow Unsigned Extensions" not available | Enable **Show features for web developers** in Safari → Settings → Advanced first. |
| Signing errors in Xcode | Select a valid development team under Signing & Capabilities. Free accounts work but require the device to be registered. |
| Extension stops working after Safari restart | With a free developer account, re-enable **Developer → Allow Unsigned Extensions** after each Safari launch. |
| iOS extension not visible | Go to Settings → Apps → Safari → Extensions on your device and toggle the extension on. |
